# Release Notes - v1.2.0

`v1.2.0` is a post-v1.1.0 closeout release for the optional EHW-5 polish rungs.
It does not broaden the main scientific claim closed at EHW-5.4a; it completes
the engineering/reveal story around that claim.

## What is new since v1.1.0

- **EHW-5.4b board-verified parameter window.** The EHW-5.4 firmware can run the
  built-in four-arm ablation table or read a PS-staged parameter block through
  the 4.6b framebuf window (`PS 0x40000000` -> `NEORV32 0xF5000000`). On board,
  one bitstream reproduced the built-in 5.4a table, then ran a staged single-arm
  `GENS=4` scan after PL logic reset, with no rebuild and no bitstream reload.
- **EHW-5.5 board-verified ICAP reveal.** A fresh no-fault baked spare-route
  island was live-edited through ICAP so the EHW-5 structural champion became
  real baked fabric INITs. Baseline truth/feature `0xe8` / `0xfbc5dabfc7`
  changed to champion truth/feature `0xa0` / `0xd2c1d02a42`, with marker `SR55`
  unchanged and no PS/NEORV32 reset.
- **Framebank loader LE mode.** `scripts/ehw2-framebank-load.py --le` stages
  little-endian parameter images generated by `scripts/ehw54-param-pack.py`;
  the default remains big-endian for EHW-2/EHW-3 framebanks.

## Non-claims

- EHW-5.5 is a reveal/instantiation proof, not a requirement for the EHW-5
  structure+weight+HW-SGD claim.
- The project still does not mutate raw Xilinx switch-matrix routing bits.
  EHW-5.5 edits LUT/select INITs inside a fixed-route island.
- EHW-4/EHW-5 metrics remain same-set deployment/adaptation metrics, not holdout
  generalization.
- The designs remain deliberately small and board-gated.

## Reproducibility Notes

- Run `tests/run_host_gates.sh` before board work.
- From EHW-5.2 onward, run `scripts/board-set-fclk50.py` before `fpga loadb`.
  Miner U-Boot can return FCLK0 to 125 MHz after reboot; EHW-5 signs off at
  50 MHz.
- For EHW-5.4b param images, use `ehw2-framebank-load.py --le` or direct U-Boot
  `mw` writes. Without `--le`, the little-endian magic `0xE5400001` appears as
  `0x010040E5`.
- For EHW-5.5, always extract frames from the fresh routed baseline/champion
  bitstreams used for that run. Target LUT sites can move between builds.

## Status

The EHW-0 -> EHW-5.5 ladder is complete through the optional reveal polish. No
mandatory ladder item remains.
