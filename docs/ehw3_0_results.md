- Fixed outer route: source pool `P=[x0,x1,x2,ZERO,ONE]`, four C1 nodes

- C1 nodes are 2-input LUT4s. `O` is a 3-input LUT8.
- Evolved fields are only LUT INITs and local mux selects:
  `logic_init[0..3]`, `init_out`, `node_sel[0..3][0..1]`, `out_sel[0..2]`.
- The physical route is fixed. No raw Xilinx routing bits are mutated.

## Validity Layer

Every mux is a pure fan-in selector, not a bus, so every genome decodes to a
single-driver circuit. Invalid selector bytes decode to harmless defaults:

- `node_sel >= 5 -> ZERO`
- `out_sel >= 4 -> A0`

The fault model is injected during evaluation and is not part of the genome.
These are modeled island faults, not physical destruction of FPGA resources.

## Interpretation

The no-fault genome implements the target majority mask `0xe8`. When
`FAULT_DISABLE_NODE(A1)` is injected, the same genome drops to `6/8` with mask
`0x88`, proving the injected fault affects the live phenotype.

The recovery run keeps the fault active and evolves a new genome that reaches
`8/8` again. The repaired output mux no longer selects `A1`; it selects `AS` as
one of the three inputs, so the repair uses the spare node and local rerouting
inside the fixed-route island.

This rung establishes the host-side decode contract for EHW-3.1. There is no
external golden cross-check yet because the spare-routing island is a new
substrate; the next rung must add the Python/C bit-exact twin.
